1. The AI Video Consistency Problem

Generative video models (e.g., Veo, Sora) face a fundamental continuity gap. While they can render photorealistic pixels, they struggle with temporal logic—leading to "melting" hands, objects vanishing behind trees, and gravity defying movements.

To build industrial-grade video workflows, we need a smart **Auditor**. The Generative AI Video Evaluator leverages specialized multi-agent critique systems to review these videos, identify timestamp-exact failures, and automatically recalculate corrected input prompts to fix the video in a closed-loop cycle.

2. The Multi-Agent Paradigm

Architectural Decision

Evaluating generative video is a high-density cognitive task. When a single foundation model (even with 1M+ context) is tasked with "finding all bugs," it suffers from **Attention Dilution**—focusing on stylistic flair instead of physical anomalies.

Our architecture isolates reasoning into parallel, specialized micro-models:

  • Object Permanence Indexer: Anchors bounding-box consistency in the model weights.
  • Physics & Motion Auditor: Audits trajectory vector plausibility.
  • Temporal Consistency Tracker: Benchmarks background noise and frequency crawl.

3. Multimodal Token Budgeting

Connecting frames to **Gemini 1.5 Pro** requires balancing context size and inference latency. We use client-side downscaling: 

// src/lib/video-utils.ts
const maxDim = 800; // Cap to 800px for optimal token weight
const scale = Math.min(maxDim / videoWidth, maxDim / videoHeight, 1);

Token Impact: Scaling to 800px caps the image token weight to roughly **258 tokens per frame**. For a 12-frame sweep, the total payload is ~3,100 tokens—ideal for fast parallel REST execution without choking the concurrency buffers.

4. The "Negative Bias" Prompting Strategy

Foundation models are trained to be helpful, chatty, and polite. In a QA environment, this leads to **Agreeable Response Syndrome** where the model overlooks subtle deformities.

We inject a "Professional Suspicion" weight into the pre-prompt payload to override this behavior:

// Implicit instruction weights
`You are a CRITICAL auditor. 
AI-videos FREQUENTLY violate basic laws of nature. 
Your role is to FLAG anything that looks digital, impossible, or glitched.`

Sensitivity Multiplier: The frontend sensitivity slider (0-100) is injected directly into the system prompt config for the agent, acting as an attention-temperature dial.

5. Forced Structured Inference (Tooling)

To feed the timeline graph, the responses *must* be deterministic. Instead of relying on raw text parsing or `json` output modes, we enforce structured inference via the **`functionCallingConfig`** setup.

// src/lib/gemini.ts
toolConfig: {
  functionCallingConfig: {
    mode: 'ANY', // FORCES the model to output a tool call
    allowedFunctionNames: [toolName] 
  }
}

By enforcing mode: 'ANY', we guarantee that the response payload will conform perfectly to the exact `FLAG_TOOL_SCHEMA` provided, completely bypassing hallucinated or wordy markdown wrappers.

6. Prompt-to-Prompt Remediation Vector

Continuous loop optimization works by converting **Qualitative Flags** back into **Quantitative Prompt Constraints**.

The Loop Mechanics:

  1. Audit: Physics Agent returns `At 00:03, hand clips table`.
  2. Stitch: The remediation pipeline maps this flag to the initial prompt.
  3. Rewrite: An optimizer agent in remediation.ts injects a negative constraint in the prompt structure: "Ensure hand preserves depth layering and does not intersect the table geometry."
  4. Re-verify: The new output is rendered by Google Veo with a STYLE reference to the original, locking down aesthetic drift while correcting geometry.

Code-Level Prompt Stitching

The system aggregates the initial context and the structural audit before passing to the Optimizer model:

// src/lib/remediation.ts
export async function generateRegenerationPrompt(
  videoName: string, flags: Flag[], originPrompt?: string
) {
  const prompt = `
    Original prompt used: "${originPrompt}"
    Detected issues by AI Evaluation Agents:
    ${flags.map(f => `- [${f.category}]: ${f.description}`)}

    Analyze the original prompt. Suggest an improved prompt for Google Veo that:
    1. Retains the original concept.
    2. Incorporates specific technical constraints to resolve the issues.
    3. Explicitly instructs the model to avoid the failed behaviors.
  `;
  
  const result = await model.generateContent(prompt);
  return result.response.text();
}

7. LLM Ops & Resiliency

Parallelizing 3 agents per video naturally stresses rate limits (HTTP 429). The system uses a **nested backoff algorithm** for reliable scaling.

// Exponential backoff in gemini.ts
if (isRateLimit) {
   const baseDelay = 3000; // Wait longer if rate limited
   await new Promise(r => setTimeout(r, baseDelay * Math.pow(2, i)));
}

Concurrency Management: State operations map over a standard Promise.all() allowing fast resolution times of under 12 seconds per video analysis pipeline.
