Developer Onboarding

Welcome to the Generative AI Video Evaluator codebase. This project is built as a High-Fidelity Analyzer capable of detecting subtle anomalies in AI-generated videos. It bridges the gap between generation models and production standards.

Primary Engine: The core logic runs client-side via React, orchestrating parallel streams to the Gemini 1.5 Pro endpoint for visual auditing.

2. Project Structure

5. Under the Hood: Key Drivers

Below are detailed explanations of the critical codebases running the analyzer.

A. useVideoAnalysis.ts (The Orchestrator)

This hook manages the isAnalyzing and progress states. It ensures all async agent pipelines are execute in parallel to reduce wait times.

// Simplified Pipeline in useVideoAnalysis.ts
const analyzeVideo = async (file: File) => {
  // 1. Frame Extraction
  const { frames, duration } = await extractFrames(file, 12);

  // 2. Parallel Execution
  const agentPromises = enabledAgents.map(async (config) => {
    const flags = await runAgent(config, frames, duration);
    return { agent: config.type, flags, status: 'complete' };
  });

  // 3. Merge Results
  const results = await Promise.all(agentPromises);
}

B. video-utils.ts (Frame Extraction)

To feed frames into Gemini, we use a client-side HTML5 Canvas approach. For performance & size limit reasons, we cap the dimensions.

// Capping Resolution in video-utils.ts
const maxDim = 800; // Max dimension for performance
const scale = Math.min(maxDim / video.videoWidth, maxDim / video.videoHeight, 1);
canvas.width = Math.round(video.videoWidth * scale);
canvas.height = Math.round(video.videoHeight * scale);
ctx.drawImage(video, 0, 0, canvas.width, canvas.height);
Performance Tip: Capping at 800px maintains sufficient quality for Gemini's vision sensors to track object details, without hitting token/size bandwidth limits.

6. Extending the Core

To add a new critique agent (e.g., Audio Consistency Agent), follow these 3 steps:

Step 1: Register Agent Type

In src/lib/types.ts, append your type to DefaultAgentType:

export type DefaultAgentType = 'object_permanence' | 'physics_motion' | 'temporal_consistency' | 'audio_sync';

Step 2: Configure the Prompt

In src/lib/agents.ts, create a new entry in DEFAULT_AGENT_CONFIGS with your system prompt instructions. Ensure you instruct the model to use the FLAG_TOOL_SCHEMA.

Step 3: Connect to Orchestrator

In src/hooks/useVideoAnalysis.ts, ensure the batch execution maps over all `enabled` agents dynamically from your configuration.

7. API Data Structures

The system relies on highly-predictable interfaces located in src/lib/types.ts.

The Flag Interface

Captures a specific anomaly on the timeline.

export interface Flag {
  id: string;
  timestamp: string;      // MM:SS format for UI Display
  timestampSeconds: number; // For accurate timeline scrub position
  severity: 'critical' | 'warning' | 'info';
  category: AgentType;
  description: string;
  confidence: number;
}

The AnalysisResult Interface

export interface AnalysisResult {
  id: string;
  videoName: string;
  videoDuration: number;
  coherenceScore: number;
  flags: Flag[];
  originPrompt?: string;  // Passed forward for the Regeneration Loop
}

8. FAQ & Troubleshooting

Common engineering issues found during integration:

  • Q: Why is "No frames could be extracted" showing up?
    A: The video codec might be unsupported by the browser's <video> element. Ensure files are in standard H.264 MP4.
  • Q: I got a 429 Error from Gemini when running batch.
    A: The parallel execution triggers multiple requests at once. The gemini.ts handler includes a retry mechanism, but repeated 429's might mean you need to increase the delay threshold or lower the number of parallel agents for large batches.
  • Q: How is the score calculated?
    A: Total score starts at 100 and applies weighted deductions: Critical is -15, Warning is -7.
