Reality Check Engine

Reality Check Engine Project Documentation

The Generative AI Video Evaluator is a high-fidelity quality assurance engine for AI-generated video. It uses a multi-agent critique model (ADK Agents) to detect physical impossibilities, object permanence violations, and environmental drift.

Introduction

As AI video generation (Sora, Veo, Kling) becomes more prevalent, the bottleneck has shifted from "creation" to "evaluation." Current AI models frequently suffer from "hallucinations"—subtle errors in physics, anatomy, and temporal continuity that the human eye often misses at first glance.

The Generative AI Video Evaluator automates this inspection process. It doesn't just "watch" a video; it breaks it into constituent frames, analyzes them semantically using distributed ADK agents, and benchmarks the results against ground truth data to ensure maximum reliability.

Core Features

1. Single Video Analysis

Users can upload a single MP4/WebM file. The engine uses canvas to extract high-resolution frames (defaulting to 12 keyframes across the duration). These frames are sent to the ADK agents for multi-dimensional critique.

2. Batch Evaluation & Coverage

Crucial for enterprise workflows, the Batch mode allows uploading 20+ videos at once. It supports Ground Truth Benchmarking, where human-labeled issues are compared against AI detections to compute a "Recall Coverage" score.

3. Real-time Flagging & Timelines

Issues are plotted on an interactive timeline. Users can jump directly to the timestamp of an issue, confirm it, or dismiss it (updating the feedback loop).

Ground Truth Benchmarking is the project's unique competitive advantage. It ensures that the AI evaluation remains honest and aligned with human standards.

Evaluation Agents

Evaluation is split into specialized domains to prevent prompt fatigue and increase accuracy.

OBJ Object Permanence Agent

Tracks identity drift via Negative Bias prompting. Powered by Gemini 1.5 Pro, it identifies the exact start and end times of facial morphing or cloth pattern changes.

PHY Physics & Motion Agent

Audits gravity, biomechanics, and collision. Detects if a person's legs bend impossibly or if objects clip through walls.

TMP Temporal Consistency Agent

Focuses on environment. Detects background warping, texture crawling, and abrupt lighting/hue shifts between frames.

Remediation & Regeneration

Detecting issues is only half the battle. The Generative AI Video Evaluator helps users *fix* the videos automatically through Closed-Loop Prompt Remediation.

Strategy Recommendation

Based on problem "density," the system suggests a strategy:

  • Cut & Download: For videos with 1-2 minor issues at specific timestamps.
  • Advanced Regeneration: For videos where problems are systemic (density > 30%).

Google Veo Integration

When "Regenerate" is chosen, the engine performs the following:

  1. Prompt Analysis: Gemini analyzes the failed original prompt + the detected flags.
  2. Prompt-to-Prompt Optimization: Gemini 1.5 Pro analyzes the original failure + the flags to generate a "Corrective Prompt" that explicitly instructs the model to avoid past failures.
  3. Generation: The new prompt is sent to Google's Veo model via Vertex AI.
  4. Verification: The new video is automatically re-evaluated by the same agents to ensure a "Clean Pass."

The Veo API Protocol (Regeneration)

The system submits a long-running operation to Vertex AI. Below is the structure of the POST payload:

{
  "instances": [{
    "prompt": "Corrected scene description...",
    "referenceImages": [{
      "image": { "bytesBase64Encoded": "..." },
      "referenceType": "STYLE"
    }]
  }],
  "parameters": {
    "aspectRatio": "16:9",
    "durationSeconds": 5
  }
}

The system identifies exact start/end times for issues, which are summarized in the prompt to guide the model.

Deployment Guide

The project is containerized for Google Cloud Run deployment.

Manual Deployment

# Deployment trigger
./deploy_gcp.sh

# This runs the following:
gcloud run deploy genai-video-eval \
  --source . \
  --region us-central1 \
  --allow-unauthenticated \
  --port 8080

Nginx Configuration

Since this is a Single Page Application (SPA), the nginx.conf is configured to handle the try_files $uri $uri/ /index.html pattern and proxy API requests.
